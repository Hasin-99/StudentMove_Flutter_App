Vercel
------
1. Create a Postgres database (Neon via Vercel integration, or Supabase/Vercel Postgres). Copy the connection string.
2. New Vercel project → set Root Directory to "admin" (this folder).
3. Environment variables (Production + Preview):
   - DATABASE_URL
   - ADMIN_SESSION_SECRET    (32+ random characters; encrypts the session cookie)
   - ADMIN_SEED_SUPER_EMAIL / ADMIN_SEED_SUPER_PASSWORD (first seed bootstrap)
   - ADMIN_SEED_TRANSPORT_EMAIL / ADMIN_SEED_TRANSPORT_PASSWORD (first seed bootstrap)
4. Deploy. Build runs: prisma migrate deploy && next build
5. Seed admin accounts + sample data (first setup): from your machine, with DATABASE_URL pointing at the same DB:
   npm run db:seed

Local development
-----------------
1. Copy .env.example to .env.local and fill all variables.
2. npx prisma migrate dev
3. npm run db:seed   (optional)
4. npm run dev → http://localhost:3000/admin/login

Flutter app
-----------
Point API_BASE_URL at this deployment origin (no trailing slash), e.g.:
  flutter run --dart-define=API_BASE_URL=https://your-app.vercel.app

The app calls GET /api/v1/schedules (CORS enabled for browser testing).
